Git操作指南

2021-03-14 #git
快速上手 git下载地址：https://git-scm.com/ 配置用户信息git config --global user.name = &lt;username&gt; # 配置用户名，加&quot;&quot; git config --global user.email = &lt;useremail&gt; # 配置邮箱，加&quot;&quot; git config --list # 查询配置信息 创建目录并初始化仓库，会生成一个.git的隐藏文件夹git init &lt;dirName&gt; 关联远程仓库git remote add &lt;name&gt; &lt;url&gt; # 添加远程仓库 git remote #列出所有远程仓库 复制远程仓库中的项目git clone &lt;url&gt; 文件提交git add &lt;file&gt; #把文件从workspace添加进index中 git commit &lt;file&gt; -m &lt;comment&gt; #把文件从index添加进repostory中 从远程仓库更新项目git pull &lt;name&gt; master 提交到远程仓库git push &lt;name&gt; master Git原理 Git工作空间 Git常用命令 查看当前状态 git status 把指定文件恢复到暂存区 git reset HEAD &lt;file&gt; 把暂存区的文件恢复并覆盖到工作区 git checkout -- &lt;file&gt; 查看提交历史 git log 文件在各个空间的切换 HEAD指针指向仓库中的当前版本：HEAD~n,表示前第n个版本； 移动HEAD指向，指向上一个快照；再将指针指向的快照回滚带暂存区（默认情况） git reset --mixed HEAD~ 移动HEAD指针的指向，指向上一个快照 git reset --soft HEAD~ 移动HEAD指针的指向，指向上一个快照；将指向的快照回滚到暂存区；将暂存区的文件还原到工作区 git reset --hard HEAD~ 版本回滚 回滚到指定快照 git log #查看快照id（一串哈希值） git reset [id] #回到该id快照 版本比较 比较工作空间和暂存目录git diff 比较两个历史快照git diff 快照id1 快照id2 比较当前工作去和仓库中的快照git diff 快照id 比较暂存区和仓库的快照(不加快照id默认是HEAD指向的版本)git diff --cached 快照ud 奥义图 其他 更正最新一次的提交备注git commit --amend -m &quot;description&quot; 删除文件（除了删除工作区域和暂存区域的该文件外，还需要修改仓库HEAD指向，将其指向上一个版本，上个版本是不包含该文件的）git rm &lt;file&gt; git reset --soft HEAD~ 删除暂存区域的文件， 不会删除工作目录的文件git rm --cached &lt;file&gt; 重命名文件git mv &lt;oldname&gt; &lt;newname&gt; Git分支 创建新分支git branch &lt;branchname&gt; --decorate:下那是tag信息； --oneline:精简显示每个快照； --graph:图形化显示git log --decorate --oneline --graph -all 切换分支，此时HEAD指针会指向切换的分支上git checkout &lt;brachname&gt; git checkout -b &lt;branchname&gt; #创建分支，并切换到新分支中 合并分支git merge &lt;branchname&gt; 删除分支git branch -d &lt;branchname&gt; ssh授权 授权命令，以生成公钥和私钥： ssh-keygen -t rsa 在github上的settings-&gt;SSH and GPG keys中添加公钥 验证是否授权成功：ssh -T git@github.com
